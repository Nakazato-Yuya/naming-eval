# TOKENIZATION_RULES — トークン化/正規化ルール

## 目的
「説明可能性」と「安定性」を両立しつつ、誤った分割を防ぐ。

## 絶対ルール
- カタカナ連続は原則1トークンとして扱う
- 例：「システム」を「シス」「テム」に分割しない
- 形態素解析器（fugashi/janome）を使う場合でも、後処理でカタカナ複合語を保護する

## 推奨処理
- 正規化：空白除去、半角全角、ひらがな→カタカナ、許可記号のみ残す
- fallback tokenization：文字種（漢字/ひら/カタ/英数）連続でチャンク化
- 英字：CamelCase / snake / hyphen を安全に分割

## 回帰テスト（必須例）
- 「システム」→ ["システム"]
- 「ジャパンシステム」→ ["ジャパンシステム"] または ["ジャパン","システム"]（過分割禁止）
- 「Tech-System」→ ["tech","system"]（意図した分割）
