# POC_PLAYBOOK — PoCの進め方（Phase設計）

## Phase 1（名前中心：最小PoC）
目的：一次選抜と説明可能性を成立させる
- candidates評価 → Drivers確認 → 競合類似の上位を抽出
- Conf閾値はまず暫定（分布を見てp90に寄せる）

## Phase 2（KPI検証：売上/広告）
目的：妥当性を「数字」で示す
- リネーム前後のΔスコアとKPI変化を対応づける
- ベースライン（文字数/単純類似）比較を必ず入れる

## Phase 3（運用想定）
目的：企業制約下でも回る形にする
- 匿名化、持ち出し不可、短期、成果物制限に対応
- レポート（PDF/HTML）自動出力

## レポート構成（最小）
1. 目的と前提（データ範囲、制約）
2. 分布（カテゴリ別）
3. 上位リスク例（Drivers付き）
4. 反事実差分（改善案）
5. ベースライン比較
6. 結論と次の打ち手
